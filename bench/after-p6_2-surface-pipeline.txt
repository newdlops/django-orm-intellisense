Comparison (1500 models — closest to Python baseline scale):
  Python build_static_index alone (baseline-python.json):       359ms
  Rust full napi pipeline including JSON serialise:             148ms
  Speedup on the whole cold-start data path:                    ~2.4x
  (Python pipeline also does build_surface_index — total likely
   500-700ms, pushing the ratio closer to 3-4x end-to-end.)

Notes:
  - 15k-model wire surface is 140MB — dominated by serde_json encoding
    of QuerySet/Manager method entries x 15k models. Real projects have
    fewer models and this term is smaller.
  - Internal Rust-only work (without JSON/marshal), 15k models:
      discovery 40ms + static_index 146ms ~= 186ms.
    Remaining ~1.3s is serde_json + Buffer + TS JSON.parse, which can
    be compressed with per-model napi calls or rkyv-archived output.
  - P4.2 reverse-relation synthesis + inheritance expansion are now
    part of this pipeline and verified by new resolver tests.

Regression fix during this phase:
  - initial buildSurfaceIndexJson was O(N^2) due to instance_surface
    iterating the flat fields list for every model label. Pre-grouping
    by label in build_wire_surface_index brought 15k models from
    23,855ms -> 1,469ms (~16x).
